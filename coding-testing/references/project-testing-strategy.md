## 测试金字塔

### 测试分层比例（v2.0 四层体系）

```
            ┌─────────────────┐
            │  L4 浏览器 E2E  │  ← 10%  跨模块核心旅程
            │  (Playwright)   │
            ├─────────────────┤
            │  L3 API 集成测试 │  ← 25%  Controller 全链路
            │ (Spring Boot)   │
            ├─────────────────┤
            │  L2 组件测试     │  ← 15%  前端组件交互
            │  (Vitest)       │
            ├─────────────────┤
            │  L1 后端单元测试 │  ← 50%  DomainService 业务逻辑
            │  (JUnit 5)      │
            └─────────────────┘
```

### 各层测试说明

| 层级 | 测试类型 | 占比 | 目标 | 框架 |
|------|---------|------|------|------|
| **L1** | 后端单元测试 | 50% | 验证 DomainService 业务逻辑 | JUnit 5 + Mockito |
| **L2** | Vitest 组件测试 | 15% | 验证前端组件交互、渲染、校验 | Vitest + Vue Test Utils |
| **L3** | API 集成测试 | 25% | 验证 Controller→AppService→DB 全链路 | Spring Boot Test + MockMvc |
| **L4** | 浏览器 E2E | 10% | 验证跨模块用户旅程 | Playwright |

### 测试分层定位

```
┌─────────────────────────────────────┐
│  L4 E2E测试: 跨模块核心旅程         │ ← Playwright（详见 e2e-testing.md）
├─────────────────────────────────────┤
│  L3 API集成测试: Controller全链路   │ ← Spring Boot Test / Testcontainers
├─────────────────────────────────────┤
│  L2 组件测试: 前端组件交互          │ ← Vitest + Vue Test Utils（新增）
├─────────────────────────────────────┤
│  L1 单元测试: DomainService        │ ← JUnit 5 + Mockito
├─────────────────────────────────────┤
│  边界测试: 极端场景与边界条件        │ ← JUnit 5 + Mockito
└─────────────────────────────────────┘
```

### 各层职责（不重复覆盖）

| 验证目标 | 归属层级 | 说明 |
|---------|---------|------|
| 业务逻辑正确性 | L1 单元测试 | DomainService |
| 后端 API 正确性 | L3 API 集成测试 | CRUD、校验、权限、异常 |
| 前端组件行为 | L2 Vitest | 表单校验、渲染、交互 |
| 跨模块集成 | L4 E2E | 用户旅程、导航、浏览器兼容 |

## 边界测试强制性要求（v2.1 风险驱动模型）

### 边界测试定义

边界测试是针对系统在极端条件、临界值、异常输入等边界情况下的测试，确保系统在各种极限场景下的稳定性和正确性。

### 风险驱动规则

> **核心理念**：边界测试数量由方法的风险等级决定，而非固定的 10 个点。高风险方法投入更多测试，低风险方法适度覆盖。

| 规则 | 说明 | 适用范围 |
|------|------|---------|
| **规则1** | 所有DomainService必须有边界测试（独立类或 @Nested 分组） | 领域层所有服务 |
| **规则2** | 按风险等级确定边界测试数量 | DomainService公共方法 |
| **规则3** | DomainService整体覆盖率≥80% | 所有测试 |
| **规则4** | 高风险方法（P0/P1）必须覆盖安全相关边界 | 安全敏感方法 |

### 风险等级与边界测试要求

| 风险等级 | 判定条件 | 边界测试数量 | 示例 |
|---------|---------|------------|------|
| **P0-关键** | 涉及数据安全/权限/资金的操作 | 8+ | 权限校验、密码加密、数据删除 |
| **P1-高** | 多分支业务逻辑、跨服务调用 | 5-8 | 规则引擎执行、任务调度 |
| **P2-中** | 单一职责的CRUD操作 | 3-5 | 字典管理、部门增删改查 |
| **P3-低** | 纯查询、无状态转换 | 1-3 | 列表查询、详情查询 |

### 风险评级决策树

```
方法是否涉及数据安全或权限？
  ├─ 是 → P0-关键
  └─ 否 → 方法是否有3个以上分支？
              ├─ 是 → P1-高
              └─ 否 → 方法是否修改数据？
                          ├─ 是 → P2-中
                          └─ 否 → P3-低
```

### 边界测试组织方式（二选一）

方式一：独立 BoundaryTest 类（推荐用于 P0/P1 方法多的服务）
```
src/test/java/com/dp/dataengine/domain/service/user/
├── UserDomainServiceTest.java          # 主测试类
└── UserDomainServiceBoundaryTest.java  # 边界测试类
```

方式二：主测试类内的 @Nested 分组（推荐用于 P2/P3 为主的服务）
```java
@Nested
@DisplayName("边界测试")
class BoundaryTests {
    @Test void shouldRejectNullUsername() { ... }
    @Test void shouldRejectEmptyUsername() { ... }
}
```

## 测试用例数量标准

### 测试用例数量公式

```
测试用例数 = 方法数 × 3 + 分支数 × 2 + 边界数 × 1
```

**示例**：

假设一个Service有5个方法，每个方法平均有3个分支，需要测试5个边界：

```
测试用例数 = 5 × 3 + 15 × 2 + 25 × 1 = 15 + 30 + 25 = 70个测试用例
```

### 按复杂度的用例要求

| 复杂度 | 说明 | 最少用例数 | 适用场景 |
|--------|------|-----------|---------|
| **简单方法** | 无分支、无边界 | 5个 | 简单CRUD |
| **中等方法** | 有分支、有边界 | 8个 | 带条件判断的CRUD |
| **复杂方法** | 多分支、多边界 | 12个 | 复杂业务逻辑 |
| **核心方法** | 核心业务逻辑 | 15个 | 关键业务方法 |

### 测试用例类型分布

| 类型 | 占比 | 说明 |
|------|------|------|
| **正常流程** | 30% | 正常业务场景 |
| **异常流程** | 30% | 异常处理场景 |
| **边界条件** | 40% | 边界与极端场景 |

## 边界测试覆盖率

### 覆盖率计算

```
边界测试覆盖率 = (已测试边界点 / 总边界点) × 100%
```

### 通用边界测试清单

每个方法必须测试的10个边界点：

| # | 边界类型 | 说明 | 示例 |
|---|---------|------|------|
| 1 | **参数为null** | 测试null参数处理 | `createUser(null)` |
| 2 | **参数为空字符串** | 测试空字符串处理 | `createUser("")` |
| 3 | **参数为最小长度** | 测试最小有效长度 | `username = "a"` |
| 4 | **参数为最大长度** | 测试最大有效长度 | `username = "a".repeat(50)` |
| 5 | **参数包含特殊字符** | 测试特殊字符处理 | `username = "test_user-123@example.com"` |
| 6 | **参数包含Unicode字符** | 测试Unicode支持 | `username = "测试用户"` |
| 7 | **参数为0或负数** | 测试数值边界 | `id = 0` 或 `id = -1` |
| 8 | **参数为极值** | 测试Long极值 | `id = Long.MAX_VALUE` |
| 9 | **参数为重复值** | 测试唯一性约束 | 创建重复用户名 |
| 10 | **参数组合边界** | 测试多个边界组合 | `null` + `""` + `0` |

### 边界测试分类

| 分类 | 测试内容 | 权重 |
|------|---------|------|
| **空值边界** | null、空字符串 | 20% |
| **长度边界** | 最小长度、最大长度 | 20% |
| **数值边界** | 0、负数、极值 | 15% |
| **ID边界** | ID为0、负数、极值 | 15% |
| **状态边界** | 所有枚举值、状态切换 | 10% |
| **特殊字符** | 特殊字符、Unicode | 10% |
| **并发边界** | 并发创建、并发更新 | 5% |
| **性能边界** | 大数据量、批量操作 | 5% |
